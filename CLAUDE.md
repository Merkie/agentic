# @merkie/agentic

Opinionated `streamText()`-first framework wrapping the Vercel AI SDK + OpenRouter with session state, JSONL persistence, queued messages, abort salvage, and cost/context tracking.

See `README.md` for the full feature list and usage examples.

## Stack

- **Runtime:** Node `>=20`, ESM-only (`"type": "module"`)
- **Language:** TypeScript
- **Package manager:** `pnpm@10.28.0` (pinned in `package.json` — do not switch to npm/yarn/bun)
- **Build:** `tsup` → `dist/`
- **Tests:** `vitest`
- **Core deps:** `ai`, `@openrouter/ai-sdk-provider`, `zod`

## Layout

- `src/` — library source
  - `index.ts` — public exports (keep the public API surface tight)
  - `session.ts` — `AgenticSession`, the main runtime around `streamText()`
  - `jsonl.ts` — append-only `.jsonl` logger + replay
  - `abort-salvage.ts` — turns partial streams into replayable `ModelMessage[]`
  - `openrouter.ts` — provider setup, model catalog, context-length lookup
  - `tools.ts` — `createTools` factory (closure-based context binding)
  - `context.ts` — context-window math from AI SDK usage
  - `types.ts` — public types
- `tests/` — unit tests (vitest)
- `examples/` — runnable demos (`tsx`)
- `dist/` — build output (gitignored, generated by `pnpm build`)

## Commands

```sh
pnpm install
pnpm typecheck      # tsc --noEmit
pnpm test           # vitest run
pnpm build          # tsup
pnpm dev:send-message -- "prompt here"   # needs OPENROUTER_API_KEY
```

## Conventions

- **JSONL persistence is the source of truth.** Every `streamText()` input and step output is appended verbatim as `ModelMessage` objects — no custom schema, no display-string round-trip. Replay is "read the lines back in order." Don't introduce reconstruction layers.
- **Session option is `file`, not `logFile`.** Renamed in commit `430b9df`.
- **Public API surface is intentionally small.** Only re-export from `src/index.ts` what consumers actually need.
- **Tools bind state via closure**, through `createTools(({ context }) => ...)`. Don't let the model control parameters that should come from app state.
- **One active run per session.** Messages sent mid-run are queued and grouped into the next turn — don't add parallel-run logic.
- **Aborts must stay replayable.** Salvage logic in `abort-salvage.ts` synthesizes tool results for dangling tool calls so the JSONL stays valid for replay. Preserve this invariant.
